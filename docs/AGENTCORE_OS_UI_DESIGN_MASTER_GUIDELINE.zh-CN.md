# 智枢 OS UI 设计总纲

这份文档用于统一智枢 OS 的 UI 方向。

从现在开始，智枢 OS 的界面设计不以其他产品为母版。
我们只根据自己的产品定位、业务目标、工作流结构和真实落地需求来设计。

## 一句话定义

智枢 OS 不是一个 AI 聊天界面，
不是一个很多 Agent 的监控台，
也不是一个普通 SaaS 后台。

智枢 OS 是一个：

**本地优先、面向真实业务、按行业与角色组织、以工作流推进、可审批、可沉淀资产的业务操作系统。**

## 设计总原则

所有 UI 设计都必须同时满足这 6 条原则：

1. 先服务真实工作，再服务视觉表达
2. 先让用户知道从哪开始，再展示系统能力
3. 先让流程可推进，再让页面显得高级
4. 先保证前后端可实现，再追求概念感
5. 先保证稳定可信，再追求炫酷效果
6. 先强化我们的产品特质，不盲目模仿其他产品

## 智枢 OS 的 UI 核心任务

UI 必须帮用户一眼看懂这些问题：

1. 我属于哪个行业场景
2. 我当前以什么角色进入
3. 我最应该启动或继续哪个工作流
4. 哪些步骤是 AI 自动执行
5. 哪些步骤需要人工审批
6. 当前系统是否稳定运行
7. 工作结果最终沉淀到了哪里

如果页面不能帮助用户回答这些问题，这个页面就没有完成任务。

## 产品心智

用户打开智枢 OS 后，应该形成这样的直觉：

- 这是我的工作主控台
- 我不是在找工具，而是在推进工作
- 系统已经按行业、角色、流程帮我组织好了
- 我能清楚地看到 AI 在做什么
- 我能清楚地知道哪些地方需要我确认
- 我的结果不会丢，会沉淀为以后可复用的资产
- 这个系统是稳定的、可信的、可长期使用的

## 我们不做什么

智枢 OS 不应该被设计成：

- 聊天机器人产品
- AI 玩具箱
- 通用项目管理工具
- 运维监控后台
- 纯 app launcher
- 炫技型黑客风面板
- 只有首页好看、深层页塌陷的概念设计

## 信息架构总纲

智枢 OS 的主信息架构必须围绕以下路径展开：

1. Industry
2. Role
3. Workflow
4. Approval
5. Asset

页面层级建议统一为：

- 一级：Home / Solutions / Roles / Workflows / Assets / Approvals / Settings
- 二级：行业 solution 页、角色 desk 页、工作流列表页、资产分类页、审批分类页
- 三级：workflow run 详情页、stage 详情页、asset detail 页、approval detail 页、customer/project detail 页

不允许只有一级页面有设计感，二级三级页面退化成普通表格后台。

## 首页设计原则

首页不是欢迎页，也不是 app 图标页。
首页必须是总控台。

首页至少要包含：

- 今日焦点
- 当前角色
- 推荐进入的行业或 solution
- 正在运行的工作流
- 待审批事项
- 最近沉淀的资产
- 本地运行状态
- 快速命令入口
- 继续未完成工作的入口
- 下一个最值得推进的动作

首页必须让用户在 3 秒内知道应该从哪里开始。

## 行业页设计原则

行业页不是分类目录，而是解决方案入口。

每个行业页必须回答：

- 这个行业最常见的目标是什么
- 这个行业最典型的工作流是什么
- 这个行业最适合哪些角色进入
- 这个行业常用哪些资产和执行模块

行业页要让用户产生“这是我的业务场景”的感觉。

## 角色工作台设计原则

角色工作台不是简单换个标题。
每个角色工作台都必须像一个专属工位。

每个角色页都应具备：

- 当前角色核心指标
- 当前优先任务
- 推荐工作流
- 常用快捷动作
- 待审批或待确认事项
- 相关资产入口

角色工作台要让用户感觉：

“这是给我这个角色准备的，不是所有人都一样。”

## 工作流页面设计原则

工作流页面是智枢 OS 的核心骨架。

工作流页必须展示：

- trigger
- runtime status
- stage status
- current blocker
- next recommended action
- human approval checkpoint
- output target
- asset landing
- related records
- execution summary

工作流页不能只是流程图。
它必须是一张“正在运行的业务对象详情页”。

## 审批页面设计原则

审批页不是消息中心。
审批页必须是决策页面。

审批页至少要有：

- 风险等级
- 来源工作流
- 上下文摘要
- AI 建议
- 人工决策入口
- 审批后影响范围
- 写回位置
- 历史记录

审批中心要让用户感觉“我是在做判断”，而不是“我在处理通知”。

## 资产页设计原则

资产库不是文件夹页面。
资产页必须体现沉淀、复用、派生。

资产页至少要有：

- 资产来源
- 资产类型
- 适用场景
- 最近使用
- 关联 workflow
- 关联客户/项目/行业
- 一键复用
- 派生版本或相关模板

资产库要像“业务成果中心”，不是简单的存档。

## 二级页与三级页设计原则

二级页和三级页必须与一级页同样精致、统一。

必须统一：

- 布局规律
- 组件系统
- 标题层级
- 色彩语言
- 状态语言
- 间距节奏
- 交互反馈

深层页面必须具备：

- 清晰上下文
- 明确返回路径
- 稳定的导航规律
- 高信息密度下的清晰层级

## 视觉风格原则

智枢 OS 的视觉气质应该是：

- 专业
- 稳定
- 克制
- 高级
- 可执行
- 有系统感

应避免：

- 通用 AI 紫色模板风
- 过度圆润和消费级可爱感
- 过度黑客风
- 过度玻璃拟态
- 只靠大渐变和发光制造高级感

建议方向：

- 深石墨、暗青、冷灰、墨绿这类稳重基底
- 有节制的强调色，用于状态和重点动作
- 强层级、强结构、强分区
- 字体要有控制感和产品气质

## 交互原则

交互必须帮助用户推进工作，而不是制造惊喜。

必须做到：

- 高频动作路径短
- 相似动作位置一致
- 支持快速继续未完成工作
- 支持快速回到上级上下文
- 命令入口醒目且高效
- 状态变化有明确反馈
- 错误、失败、重试可理解

不要做：

- 只能靠 hover 才看见的重要信息
- 复杂难实现的炫技动画
- 打断工作节奏的夸张交互

## 稳定性表达原则

智枢 OS 必须在 UI 上体现“稳定运行”。

建议界面可见这些对象：

- 本地运行状态
- sidecar / connector 状态
- 最近同步时间
- 工作流运行状态
- 错误与恢复入口
- 审批等待状态
- 数据写回确认

这些信息必须：

- 可见但不打扰
- 专业但不工程化过头
- 让用户安心，而不是让用户紧张

## 可实现性原则

任何 UI 方案都必须满足：

- 可以映射到真实前端组件结构
- 不破坏现有多窗口桌面壳逻辑
- 不破坏工作流、审批、资产、行业页、角色页的功能结构
- 可以稳定运行
- 不为了视觉牺牲信息清晰度和操作效率

判断标准不是“是否好看”，而是：

**是否能真实落地，并让系统更强。**

## 设计评审标准

以后任何新 UI 方案，都可以用下面的问题评审：

1. 这是不是智枢 OS，而不是别的产品？
2. 用户能否快速知道从哪里开始？
3. 行业、角色、工作流三层主线是否清楚？
4. AI / Human 边界是否清楚？
5. 稳定运行感是否可见？
6. 资产沉淀是否可见？
7. 二级页和三级页是否依然统一且高级？
8. 是否能落地到真实前后端？
9. 是否比当前版本更易操作？
10. 是否在美观和效率之间做到了平衡？

## 最终目标

智枢 OS 的 UI 最终应该让人感觉：

- 这不是一个 AI 工具集合
- 这不是一个聊天页面
- 这不是一个监控台
- 这是一个真正能推进业务工作的操作系统

## 一段可直接对外使用的话

请为智枢 OS 设计一套真正属于它自己的 UI，不参考其他产品作为母版。智枢 OS 是一个本地优先、按行业与角色组织、以工作流推进、可审批、可沉淀资产的业务操作系统。设计必须首先服务真实工作，帮助用户快速理解自己所在的行业场景、当前角色、下一步工作流、AI 与人工的边界、系统运行状态，以及结果会沉淀到哪里。所有一级、二级、三级页面都必须统一、高级、可实现，并且不能为了视觉牺牲功能可用性和稳定运行。
