# AgentCore OS v1.1.1 对外发布文案

这份文案用于 GitHub Release 正文、社媒发布、官网更新、微信群/社群转发与合作方同步。

这一版的核心口径不是“新增很多功能”，而是：

- 把现有稳定线修得更准
- 把公开发布文档讲得更清楚
- 把国内外双仓入口整理得更一致

不要使用的口径：

- 不要把维护版说成大版本重构
- 不要宣传成已经完全无人化
- 不要承诺安装包分发已经恢复

当前最稳的对外定位是：

- 本地优先 AI 工作底座
- 面向真实工作流程，不只是聊天
- 销售、客服、知识资产沉淀三条高频闭环继续稳定推进
- `v1.1.1` 是对当前稳定线的维护收口版

## 1. 官方发布主文案

### 1.1 正式版

**AgentCore OS v1.1.1 正式发布。**

这是一版维护更新。

我们没有继续往系统里堆更多页面或角色，而是优先把已经进入稳定线的部分修得更可靠。

在 `v1.1.1` 中，重点完成了三件事：

- 修正 `Knowledge Vault` 流程资产一键复用的计数时机
- 修正公开发布文档对兼容迁移状态的描述
- 补齐 GitHub 和 CNB 双入口的中文发布说明

如果你更看重稳定、准确、可审计、可沉淀，而不是每个版本都追求看起来更花哨，那么 `v1.1.1` 会比 `v1.1.0` 更适合作为当前默认体验入口。

## 6. 对外 FAQ 口径

### Q1：这一版最大的变化是什么？

最大的变化是三点：

- 流程资产复用计数更准确
- 公开发布文档更准确
- GitHub / CNB 双入口更统一

### Q2：是不是大版本升级？

不是。

这是一次维护版发布，重点是修正稳定线已有问题，而不是扩展产品边界。

### Q3：适合谁？

适合：

- 想从当前稳定线开始体验的个人用户
- 需要中文入口和国内镜像说明的团队
- 正在评估销售、客服和知识资产闭环的业务方

### Q4：安装方式有变化吗？

没有。

当前默认安装方式仍然是：

- 命令行安装
- 从源码运行
