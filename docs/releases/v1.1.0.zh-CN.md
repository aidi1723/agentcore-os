# AgentCore OS v1.1.0 版本说明

`v1.1.0` 是 AgentCore OS 在首个稳定公开版本之后的下一条稳定发布线。

这一版的核心判断很明确：

AgentCore OS 不应该靠堆很多松散的 Agent 角色来证明自己，而应该靠稳定、可复核、可沉淀、可复用的真实工作流来建立价值。

## 版本定位

相对于 `v1.0.0`，这一版主要往前推进了三件事：

1. 把销售和客服两条高频链路做得更深
2. 引入少量高价值的专家角色白名单，而不是角色泛滥
3. 把 `Knowledge Vault` 从结果存放页，推进成真正可复用的流程资产层

这让 `v1.1.0` 更适合：

- 高频销售跟进
- 高频客服处理和 FAQ 沉淀
- 需要明确人工审核边界的团队
- 重视稳定积累而不是花哨自动化演示的本地优先用户

## 本版带来的核心能力

### 1. 收敛的数字员工层

当前版本新增并启用了严格收敛的专家角色白名单：

- `sales_qualification_specialist`
- `outreach_draft_specialist`
- `support_reply_specialist`
- `reality_checker`
- `knowledge_asset_editor`

这些角色不是独立漂浮在系统里的“万能 Agent”，而是绑定在具体流程阶段中使用。目标是让输出更稳定、更准确、更容易人审。

### 2. 销售与客服高频闭环更完整

当前最成熟的流程深度集中在：

- `Deal Desk`
- `Email Assistant`
- `Personal CRM`
- `Support Copilot`
- `Knowledge Vault`

销售链路已经可以完成：

- 询盘录入
- 资格判断
- 跟进草稿生成
- 人工审核
- CRM 收口
- 销售资产沉淀与复用

客服链路已经可以完成：

- 工单录入
- 建议回复生成
- `Reality Checker` 风险复核
- 人工审核
- FAQ / 升级边界资产沉淀
- 知识资产复用

### 3. Knowledge Vault 进入流程资产层

`Knowledge Vault` 现在支持结构化流程资产：

- 搜索
- 状态过滤
- 归档 / 恢复
- 删除
- 回到来源
- 复用计数
- 一键复用
- 标题、适用场景、标签、正文编辑
- 来源链路和时间戳追溯

这意味着系统不再只是“生成一段结果”，而是开始具备：

- 产出资产
- 存储资产
- 编辑资产
- 审计资产
- 在下一轮流程中复用资产

### 4. 人工审核边界保持明确

这一版仍然坚持保守的产品边界：

- 专家角色采用白名单机制
- `Reality Checker` 作为 review 节点约束层
- 关键输出仍然默认需要人工审核
- 资产沉淀发生在真实流程收口节点，而不是后台无约束乱生成

## 安装方式

当前推荐安装方式：**命令行安装 / 从源码运行**

```bash
git clone https://github.com/aidi1723/agentcore-os.git
cd agentcore-os
npm install
npm run dev
```

推荐环境：

- Node.js `20+`
- npm
