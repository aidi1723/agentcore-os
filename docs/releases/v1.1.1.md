# AgentCore OS v1.1.1

`v1.1.1` is a maintenance release for the current stable AgentCore OS line.

It does not expand the product boundary or add a new workflow surface.
Its purpose is to make the existing `v1.1.x` line more trustworthy for public distribution and daily use.

## Release positioning

Compared with `v1.1.0`, this release focuses on three maintenance goals:

1. correct workflow metrics where the old behavior could overcount reuse
2. align public documentation with the real compatibility boundary in the codebase
3. make GitHub and CNB release guidance consistent for external users

This makes `v1.1.1` a better default recommendation for:

- teams evaluating the current sales and support workflow line
- users who need clearer release documentation and installation entrypoints
- operators who want a conservative maintenance baseline instead of a feature-heavy jump

## What this release fixes

### 1. Knowledge Vault reuse metrics

`Knowledge Vault` no longer increments process-asset reuse count too early during one-click reuse.

The metric now stays aligned with actual workflow confirmation instead of a premature UI action.

### 2. Public release wording

Public release docs now describe the `openclaw` to `runtime` cleanup accurately:

- the migration is partial
- compatibility is intentionally preserved
- the public docs no longer overstate the cleanup as complete

### 3. GitHub / CNB distribution alignment

Chinese release-facing docs now include both distribution entrypoints:

- GitHub
- CNB

This reduces ambiguity for domestic users and keeps the public release path easier to follow.

## What stays the same

`v1.1.1` keeps the current stable product posture:

- local-first operation
- BYOK / API Key driven setup
- human review boundaries for critical output
- sales, support, and knowledge-asset accumulation as the most mature workflow depth

## Verification performed for this release

The release path verified in the current workspace for `v1.1.1` is:

- `npm run lint`
- `npm run build`

No broader automated end-to-end UI claim is made in this maintenance note.
