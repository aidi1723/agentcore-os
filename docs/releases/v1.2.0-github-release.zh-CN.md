# AgentCore OS v1.2.0

AgentCore OS `v1.2.0` 正式发布。

这一版不是去扩底层基础设施叙事，而是把当前最重要的产品主线继续做深：
本地优先工作台、工作流编排、连接器、发布链路和应用间状态流转。

## 本版重点

- 把共享推荐契约从 Hero Workflow 面板扩展到 6 个主要工作流执行面
- 把工作流关联任务和下一步动作提升为一等推荐信号
- 继续收口桌面壳执行历史、运行时状态 API 与发布配置覆盖面
- 全量同步当前推荐版本号到 `v1.2.0`

## 具体升级

### 1. 工作流执行面开始共享同一种推荐结构

当前版本已把共享推荐结构扩展到：

- `Deal Desk`
- `Support Copilot`
- `Deep Research Hub`
- `Content Repurposer`
- `Inbox Declutter`
- `Morning Brief`

这意味着推荐结果、下一步动作和流程反馈开始在更多主应用里使用同一种结构，而不是继续分散成多套私有返回体。

### 2. 任务与下一步动作更容易闭环

`v1.2.0` 把工作流关联任务提升为核心状态信号之一。

结果是：

- 推荐不只是说明，更能指向动作
- 多个应用之间更容易共享执行后反馈
- 销售、客服、研究、创作链路的闭环更容易持续沉淀

### 3. 稳定线继续补齐运行时与发布链路

当前版本也继续补了几块稳定性基础：

- 桌面壳执行历史与统一执行器记录更一致
- Python sidecar 对运行时状态 / 执行器历史 API 的支持更完整
- 发布账号配置与现有发布队列支持的平台更一致
- 当前稳定线已有更明确的 `test:stability` 门禁

## 已验证

本次发布前已验证：

- `npm run test:stability`

## 当前边界

`v1.2.0` 仍然强调：

- 本地优先
- BYOK / API Key 驱动
- 关键输出保留人工审核边界
- 工作流闭环、资产沉淀、连接器和发布链路是当前主线

当前不建议宣传为：

- 自研操作系统
- 自研数据库或编译器
- 完整无人公司
- 所有行业流程都已成熟覆盖

## 安装方式

当前推荐安装方式仍然是：**命令行安装 / 从源码运行**

主仓库 GitHub：

```bash
git clone https://github.com/aidi1723/agentcore-os.git
cd agentcore-os
npm install
npm run dev
```

国内镜像 CNB：

```bash
git clone https://cnb.cool/aidiyangyu/agentcore-os
cd agentcore-os
npm install
npm run dev
```

推荐运行环境：

- Node.js `20+`
- npm
