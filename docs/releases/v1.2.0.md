# AgentCore OS v1.2.0

`v1.2.0` is a depth release for the current stable AgentCore OS line.

It does not shift the product toward building a custom operating system, database, or inference substrate.
Instead, it continues the current product direction: deepen the local-first workspace, workflow orchestration, connectors, release path, and cross-app state flow until they form a more reliable loop.

## Release positioning

Compared with `v1.1.1`, this release focuses on three upgrades:

1. expanding the shared recommendation contract from Hero workflow panels into the main workflow execution surfaces
2. promoting workflow-linked tasks and next-step actions into first-class state signals
3. tightening runtime parity, publish configuration coverage, and the documented stability gate

This makes `v1.2.0` a better default recommendation for:

- teams validating coordinated sales, support, research, and creator workflows
- users who need recommendations, tasks, and workflow assets to move across apps with less shape drift
- product teams that want to deepen the current loop instead of broadening into lower-level infrastructure bets

## What this release adds

### 1. Shared recommendation structure across workflow surfaces

The shared recommendation structure previously centered on Hero workflow panels.

In `v1.2.0`, that contract now also covers the main workflow execution surfaces for:

- `Deal Desk`
- `Support Copilot`
- `Deep Research Hub`
- `Content Repurposer`
- `Inbox Declutter`
- `Morning Brief`

This reduces private per-app result shapes and gives the current workflow line a more consistent way to render recommended actions and next-step guidance.

### 2. Tasks and next steps as first-class signals

Workflow-linked tasks are now part of the core recommendation and state flow instead of being treated as a secondary afterthought.

That matters because:

- recommendations can point to concrete follow-up actions, not only explanatory text
- workflow surfaces can route users into the next action more consistently
- sales, support, research, and creator chains can reuse a shared execution-feedback shape

### 3. Runtime and release-path tightening

`v1.2.0` also continues to reduce gaps in the stable public line:

- desktop-shell execution aligns with unified executor history
- Python sidecar parity is stronger for runtime state and executor-history APIs
- publish-account settings better match the platforms already supported by the publish queue and dispatch layer
- the public stable line now has a clearer documented stability gate

## What stays the same

`v1.2.0` keeps the current stable product posture:

- local-first operation
- BYOK / API Key driven setup
- human review boundaries for critical output
- workflow depth, connectors, publish-chain handoff, and asset accumulation as the main product focus

This release should not be described as:

- a custom operating system effort
- a custom database or compiler initiative
- a fully autonomous company stack

## Verification performed for this release

The release path verified in the current workspace for `v1.2.0` is:

- `npm run test:stability`

That covers:

- core workflow regression
- publish regression
- lint
- build

No broader automated end-to-end UI claim is made in this release note.
