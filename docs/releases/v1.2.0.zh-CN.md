# AgentCore OS v1.2.0 版本说明

`v1.2.0` 是当前稳定线的一次能力深化版发布。

这一版没有把产品重心转去自研操作系统、数据库或推理底座。
它继续沿着当前主线推进：把本地优先工作台、工作流编排、连接器、发布链路和应用间状态流转做深、做稳、做成闭环。

## 版本定位

相对于 `v1.1.1`，这一版主要完成三件事：

1. 把共享推荐契约从 Hero Workflow 面板扩展到主要工作流执行面
2. 把工作流关联任务和下一步动作提升为一等状态信号
3. 把桌面壳、运行时状态、发布配置和稳定性门禁继续收口

这让 `v1.2.0` 更适合：

- 想验证销售、客服、研究、创作四类工作流协同的团队
- 需要让推荐结果、待办任务和流程资产在多个应用之间持续流转的使用者
- 希望先把现有闭环做深，而不是引入更大底层基础设施范围的产品团队

## 本版重点

### 1. 工作流推荐契约扩展到主执行面

此前共享推荐结构主要用于 Hero Workflow 面板。

在 `v1.2.0` 中，这套共享契约已扩展到主要工作流执行面，包括：

- `Deal Desk`
- `Support Copilot`
- `Deep Research Hub`
- `Content Repurposer`
- `Inbox Declutter`
- `Morning Brief`

这意味着这些应用在展示推荐动作、下一步建议和流程结果时，开始使用一致的结构化返回体，而不是继续各自维护私有结果形状。

### 2. 任务与下一步动作成为一等信号

当前版本把工作流关联任务提升为推荐系统和应用状态里的核心信号之一。

这带来几个直接结果：

- 推荐结果不再只是“给一段建议文字”
- 工作流面可以更稳定地把用户路由到下一步动作
- 销售、客服、研究、创作等链路更容易共享同一种执行后反馈结构

对于当前产品路线，这比继续扩很多孤立页面更重要，因为它直接加强了流程闭环能力。

### 3. 运行时与发布链路继续收口

`v1.2.0` 同时继续补齐当前稳定线的运行时一致性：

- 桌面壳执行链路继续对齐统一执行历史
- Python sidecar 对运行时状态和执行器历史 API 的等价支持更完整
- 发布账号配置覆盖面与现有发布队列 / 分发层更一致
- 当前公开稳定线已有更明确的稳定性门禁

## 保持不变的产品判断

`v1.2.0` 仍然坚持这些边界：

- 本地优先
- BYOK / API Key 驱动
- 关键输出保留人工审核边界
- 当前重点仍是工作台、工作流、连接器、发布链路和知识 / 流程资产沉淀

当前不建议把这一版宣传为：

- 自研底层操作系统
- 自研数据库或编译器路线
- 完整无人公司
- 全行业流程都已成熟覆盖

## 安装方式

当前推荐安装方式仍然是：**命令行安装 / 从源码运行**

```bash
git clone https://github.com/aidi1723/agentcore-os.git
cd agentcore-os
npm install
npm run dev
```

国内镜像：

```bash
git clone https://cnb.cool/aidiyangyu/agentcore-os
cd agentcore-os
npm install
npm run dev
```

推荐环境：

- Node.js `20+`
- npm
